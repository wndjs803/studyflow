# [사이드 프로젝트] 개발 기획서: StudyFlow (스터디플로우) 2.0
**목차 이미지 OCR 추출 + 자유로운 수동 편집 기능이 포함된 하이브리드 플래너**

---

## 1. 프로젝트 개요
*   **프로젝트 이름**: StudyFlow (스터디플로우)
*   **한 줄 설명**: OCR 목차 추출과 수동 To-do 관리가 결합된 스마트 학습 스케줄러
*   **프로젝트 목적**: 이미지 추출로 뼈대를 잡고, 사용자가 직접 세부 일정을 추가/수정하여 완벽한 개인 맞춤형 계획 수립
*   **개발 기간**: 4주
*   **개발 인원**: 1인 (Full-stack)

---

## 2. 문제 정의 (Problem)
*   **OCR의 한계**: 이미지 추출만으로는 내가 따로 공부하고 싶은 보충 자료나 개인적인 일정을 넣기 어려움.
*   **일반 To-do 앱의 번거로움**: 수백 페이지 분량의 책 목차를 일일이 손으로 입력하는 것은 중노동임.
*   **수정의 어려움**: 한 번 짜인 계획 중간에 새로운 할 일을 끼워 넣으면 전체 날짜를 다시 계산해야 함.

---

## 3. 해결 방법 (Solution)
*   **하이브리드 입력**: OCR로 대량의 데이터를 한 번에 넣고, 필요한 곳에 언제든 새로운 할 일을 수동으로 추가.
*   **유연한 편집 인터페이스**: 모든 항목은 인라인 수정이 가능하며, 드래그나 버튼으로 순서와 계층(들여쓰기)을 변경.
*   **스마트 리스케줄링**: 수동으로 추가한 항목도 완료하지 못하면 자동으로 다음 날로 밀어주는 로직 적용.

---

## 4. 타겟 사용자 (User)
*   교재 내용뿐만 아니라 인강, 기출문제 풀이 등 추가 일정을 함께 관리하고 싶은 수험생.
*   이미지 추출 결과를 바탕으로 본인만의 커스텀 커리큘럼을 짜고 싶은 독학러.

---

## 5. 핵심 기능 (Core Features)

| 기능 | 설명 |
| :--- | :--- |
| **OCR 및 수동 추가** | 이미지 추출 결과 리스트 하단/중간에 언제든 새로운 행(Row) 추가 가능 |
| **계층형 편집기** | 들여쓰기(Tab/Button)를 통해 항목 간 부모-자식 관계 설정 및 수정 |
| **인라인 실시간 편집** | 일반 To-do 앱처럼 클릭 즉시 텍스트 수정 및 삭제 가능 |
| **자동 일정 생성** | 전체 항목(추출+수동)을 설정한 기간/분량에 맞춰 날짜별 자동 배분 |
| **일정 자동 밀기** | 미완료된 수동 추가 항목 포함, 모든 일정을 미완료 시 다음 날로 이월 |
| **데일리 체크 뷰** | 특정 날짜를 클릭하여 해당 날짜의 할 일 확인 및 체크박스 관리 |

---

## 6. 사용자 흐름 (User Flow)
```
[시작 방식 선택] 
  → A: 이미지 업로드 (OCR 추출)
  → B: 빈 목록으로 시작 (수동 입력)
[목차 편집 단계]
  → 추출된 텍스트 수정 + 중간에 새로운 학습 항목(To-do) 직접 추가 
  → 들여쓰기로 계층 구조 정리
[일정 생성]
  → 기간/분량 설정 후 생성
[대시보드]
  → 날짜별 일정 확인 및 추가/수정/체크
```

## 7. 화면 설계 (UI / Wireframe)
*   **Editor Page**: 
    *   상단에 [행 추가] 버튼 배치.
    *   각 행 마우스 오버 시 [+][x] 버튼 노출 (특정 위치에 삽입 및 삭제).
    *   모든 텍스트 영역은 `contenteditable` 또는 `input`으로 실시간 수정 가능.
*   **Dashboard**:
    *   날짜별 리스트 하단에 "오늘의 할 일 즉시 추가" 입력창 제공.

## 8. 기술 스택 (Tech Stack)
*   **Frontend**: React, TypeScript, Tailwind CSS, **Zod** (데이터 유효성 검사), **Zustand** (편집 상태 관리), **React Query**
*   **Backend**: Spring Boot, Spring Data JPA, Spring Security
*   **AI/OCR**: Google Vision API
*   **Database**: PostgreSQL

### 💡 추가 제언 (개발 포인트)

1.  **Zod 활용**: 
    사용자가 직접 추가한 데이터는 빈 값이거나 너무 길 수 있습니다. 
    ```typescript
    const ToCItemSchema = z.object({
      content: z.string().min(1, "할 일을 입력해주세요.").max(100, "100자 이내로 작성해주세요."),
      depth: z.number().min(0).max(2),
    });
    ```
    이런 식으로 검증 로직을 넣어두면 DB 에러를 미연에 방지할 수 있습니다.

2.  **편집기 상태 관리 (Zustand)**: 
    추출된 데이터와 수동 데이터를 하나의 배열(Array)로 관리하고, `index`를 기반으로 특정 위치에 항목을 끼워 넣는 함수(`addItemAt(index)`)를 구현하시면 편리합니다.
