# [StudyFlow] 요구사항 정의서 및 상세 기능 명세서

## 1. 요구사항 정의 (Requirements)

### 1.1 사용자 요구사항 (User Requirements)
*   **R-01 (자동화)**: 사용자는 목차 이미지를 업로드하여 텍스트를 일일이 타이핑하는 수고를 덜어야 한다.
*   **R-02 (편집 자유도)**: 사용자는 추출된 내용 외에도 본인이 원하는 학습 항목을 자유롭게 추가, 수정, 삭제할 수 있어야 한다.
*   **R-03 (계층화)**: 사용자는 학습 항목 간의 우선순위나 포함 관계를 들여쓰기를 통해 직관적으로 설정할 수 있어야 한다.
*   **R-04 (지능형 배분)**: 사용자는 특정 종료 날짜나 하루 학습량을 설정하여 자동으로 일정이 분배되길 원한다.
*   **R-05 (유연한 대응)**: 사용자는 계획을 지키지 못했을 때 시스템이 자동으로 남은 일정을 조정해 주길 원한다.

### 1.2 시스템/비기능 요구사항 (System/Non-Functional)
*   **NR-01 (데이터 검증)**: 모든 입력 데이터는 TypeScript와 Zod를 통해 유효성 검사(빈 값, 글자 수 제한 등)를 거쳐야 한다.
*   **NR-02 (상태 관리)**: 편집 중인 데이터는 Zustand를 통해 클라이언트 측에서 실시간으로 동기화되어야 한다.
*   **NR-03 (반응형)**: 편집기는 데스크톱에 최적화하되, 일일 체크 대시보드는 모바일에서도 편리하게 사용 가능해야 한다.

## 3. UI/UX 컴포넌트 설계 가이드 (Figma 참조용)

### [화면 A] 스마트 편집기 (Editor)
1.  **Toolbar**: [행 추가], [저장], [초기화] 버튼.
2.  **Main List Area**:
    *   각 Row 구성: `[Drag Handle]` `[Depth Control Buttons]` `[Editable Text Input]` `[Delete Button]`
    *   Hover 상태: 해당 행을 삭제하거나 중간에 새 행을 삽입할 수 있는 버튼 노출.

### [화면 B] 대시보드 (Dashboard)
1.  **Date Picker Bar**: 가로로 긴 날짜 리스트. 오늘 날짜 하이라이트.
2.  **Task Card**:
    *   부모 항목(대/중분류) 정보가 상단에 작게 표시되고, 그 아래에 소분류 텍스트와 체크박스 배치.
3.  **Reschedule Alert**: "어제 못 한 공부 2개가 오늘로 추가되었습니다."와 같은 알림 배너.

## 4. 데이터 검증 스키마 (Zod 예시)

```typescript
import { z } from 'zod';

// 단일 항목 검증
export const ItemSchema = z.object({
  content: z.string().min(1, "내용을 입력해주세요.").max(100, "내용이 너무 깁니다."),
  depth: z.number().int().min(0).max(2),
  sort_order: z.number().int(),
});

// 전체 프로젝트 저장 시 검증
export const ProjectSchema = z.object({
  title: z.string().min(1, "교재 제목을 입력해주세요."),
  goal_type: z.enum(["PERIOD", "QUANTITY"]),
  target_value: z.number().positive(),
  start_date: z.string(), // ISO Date String
  items: z.array(ItemSchema).min(1, "최소 하나 이상의 학습 항목이 필요합니다."),
});
```
